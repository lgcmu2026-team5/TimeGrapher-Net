## G01 Watch-Position Testing

| ID | Grade | Requirement | 확인 결과 |
|---|---|---|---|
| FR-01-01 | mandatory | 시계 자세 측정 시험은 사용자에게 표준 측정 자세에서 기계식 시계를 테스트할 수 있는 기능을 제공해야 한다. | 🟢 GUI 하단에 표기됨 |
| FR-01-02 | mandatory | 시계 자세 측정 시험은 GUI를 통해 사용자에게 현재 시계 자세를 제공해야 한다. | 🟡 Main UI에서 바로 제어하는 화면은 없으며, Position 에서 설정 가능함 |
| FR-01-03 | mandatory | 시계 자세 측정 시험은 수평 자세인 CH 및 CB와 수직 자세인 6H, 9H, 3H, 12H로 구성된 표준 자세 세트를 사용자에게 제공해야 한다. | 🟡 Main UI에서 바로 제어하는 화면은 없으며, Position 에서 설정 가능함|
| FR-01-04 | recommended | 시계 자세 측정 시험은 사용자에게 중간 자세에 대한 지원을 제공하는 것이 권장된다. | 🟡 Play 중 자세 변경시 전체 GUI에 대해 재시작에 대한 룰 설정 필요 |
| FR-01-05 | mandatory | 시계 자세 측정 시험은 측정이 진행되는 동안 활성 측정 자세를 사용자에게 명확하게 표시해 주어야 한다. | 🟢 GUI 하단에 표기됨 |
| FR-01-06 | mandatory | 시계 자세 측정 시험은 활성 측정 자세와 관련된 측정 결과를 사용자에게 제공해야 한다. | 🟢 GUI 하단에 표기됨 |

## G04 Multi-Position Sequence Display

| ID | Grade | Requirement | 확인 결과 |
|---|---|---|---|
| FR-04-01 | mandatory | 다자세 시퀀스 디스플레이는 GUI 내에 시퀀스 표시 화면을 사용자에게 제공해야 한다. | 🟢 Position Tab 구현 확인함 |
| FR-04-02 | mandatory | 다자세 시퀀스 디스플레이는 여러 시계 테스트 자세에 걸친 완전한 측정 사이클을 사용자에게 제공해야 한다. | 🟢 Position Tab 구현 확인함 |
| FR-04-03 | mandatory | 다자세 시퀀스 디스플레이는 단일 시퀀스 내에서 최대 10개의 테스트 자세에 대한 결과를 캡처하고 검토할 수 있는 기능을 사용자에게 제공해야 한다. | 🟢 Position Tab 구현 확인함 |
| FR-04-04 | mandatory | 다자세 시퀀스 디스플레이는 최소한 CH, CB, 6H, 9H, 3H, 12H 자세에 대한 지원을 사용자에게 제공해야 한다. | 🟢 Position Tab 구현 확인함 |
| FR-04-05 | mandatory | 다자세 시퀀스 디스플레이는 각 측정 자세에 대한 일오차(rate), 진폭(amplitude), 비트 에러(beat error)를 사용자에게 제공해야 한다. | 🟢 Position Tab 구현 확인함 |
| FR-04-06 | mandatory | 다자세 시퀀스 디스플레이는 모든 유효한 측정 자세의 평균값인 X를 사용자에게 제공해야 한다. | 🟢 Position Tab 구현 확인함 |
| FR-04-07 | mandatory | 다자세 시퀀스 디스플레이는 가장 큰 측정값과 가장 작은 측정값의 차이인 D를 사용자에게 제공해야 한다. | 🟡 Position Tab에 구현 되어 있으나, 정확한 값은 확인 필요함|
| FR-04-08 | recommended | 다자세 시퀀스 디스플레이는 수직 자세와 수평 자세 간의 비교 기능을 사용자에게 제공하는 것이 권장된다. | 🟡 Position Tab에 구현 되어 있으나, 정확한 값은 확인 필요함 |
| FR-04-09 | recommended | 다자세 시퀀스 디스플레이는 밸런스 휠(balance-wheel)의 불균형 가능성을 파악 가능한 표시기(indicator)를 사용자에게 제공하는 것이 권장된다. | 🔴 확인 불가 / 범위 벗어남 / 신호 불량 / 측정 불가(시계위치 이동등) 에 대한 검토 필요함 |

## G07 Long-Term Performance Graph

| ID | Grade | Requirement | 확인 결과 |
|---|---|---|---|
| FR-07-01 | mandatory | 장기 성능 그래프는 장기간에 걸쳐 시계의 일오차 기록을 사용자에게 제공해야 한다. | 🟢 Long-term Rate s/d 구현 확인함 |
| FR-07-02 | mandatory | 장기 성능 그래프는 장기간에 걸쳐 시계의 진폭 기록을 사용자에게 제공해야 한다. | 🟢 Long-term Amplitude 구현 확인함 |
| FR-07-03 | mandatory | 장기 성능 그래프는 장기간에 걸쳐 시계의 비트 에러 기록을 사용자에게 제공해야 한다. | 🟢 Long-term Beat error구현 확인함 |
| FR-07-04 | mandatory | 장기 성능 그래프는 시간에 따른 일오차 변화를 사용자에게 제공해야 한다. | 🟡 시간단위 기록 확인 필요, 평균값은 계산 확인함 |
| FR-07-05 | mandatory | 장기 성능 그래프는 시간에 따른 진폭 변화를 사용자에게 제공해야 한다. | 🟡 시간단위 기록 확인 필요, 평균값은 계산 확인함 |
| FR-07-06 | mandatory | 장기 성능 그래프는 시간에 따른 비트 에러 변화를 사용자에게 제공해야 한다. | 🟡 시간단위 기록 확인 필요, 평균값은 계산 확인함  |
| FR-07-07 | recommended | 장기 성능 그래프는 테스트 진행 중 그래프의 주기적인 업데이트를 사용자에게 제공하는 것이 권장된다. | 🟢 현재 실시간 Update 됨 |
| FR-07-08 | recommended | 장기 성능 그래프는 전체 평균값을 사용자에게 제공하는 것이 권장된다. | 🟢 Long-term 하단 구현 확인함 |
| FR-07-09 | recommended | 장기 성능 그래프는 변동 범위에 대한 시각적 표시를 사용자에게 제공하는 것이 권장된다. | 🟡 노이즈 신호로 추가 확인 필요함  |
| FR-07-10 | recommended | 장기 성능 그래프는 장시간 테스트에 대한 지원을 사용자에게 제공하는 것이 권장된다. | 🟡 추가 확인 필요  |
| FR-07-11 | recommended | 장기 성능 그래프는 경과 시간이 늘어남에 따라 업데이트 빈도를 줄여 사용자에게 제공하는 것이 권장된다. | 🟡 추가 확인 필요 |

* Long-term 시간 축 정보 변경 필요 함, 수평축인 시간 축은 각 그래프 동기화 하여 보여줄 필요 있음, 수직축은 각 그래프 별 범위 설정 되도록 수정 필
