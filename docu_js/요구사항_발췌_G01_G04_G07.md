## G01 · 시계 자세(Test Position) 테스트

### 개요 (드래프트 문서)

시스템은 기계식 시계를 모든 표준 측정 자세에서 테스트할 수 있어야 하며, 현재 자세를 GUI에 식별·표시해야 한다. 자세 세트는 Chronoscope X1(G3) 매뉴얼(참고문헌 c)에 제시된 시계 자세 규약을 따르되, 수평 자세 CH·CB와 수직 자세 6H·9H·3H·12H를 포함하고, 사용 시 중간 자세(intermediate position)도 지원해야 한다. GUI는 측정이 진행되는 동안 현재 활성 테스트 자세를 명확히 표시하여, 사용자가 화면에 표시되는 결과가 어떤 방향과 연관된 것인지 항상 알 수 있도록 해야 한다.

> *참고: NIHS 95-10 / ISO 3158에 따른 테스트 자세 표기*

### 기능 요구사항 (FR)

| ID | 등급 | 요구사항 |
|----|------|----------|
| FR-01-01 | 필수 | 시계 자세 테스트는 사용자에게 기계식 시계를 표준 측정 자세에서 테스트할 수 있는 기능을 제공해야 한다(shall). |
| FR-01-02 | 필수 | 시계 자세 테스트는 사용자에게 GUI에서 현재 시계 자세를 표시해야 한다(shall). |
| FR-01-03 | 필수 | 시계 자세 테스트는 사용자에게 수평 자세 CH·CB, 수직 자세 6H·9H·3H·12H로 구성된 표준 자세 세트를 제공해야 한다(shall). |
| FR-01-04 | 권장 | 시계 자세 테스트는 사용자에게 중간 자세(intermediate position) 지원을 제공하는 것이 바람직하다(should). |
| FR-01-05 | 필수 | 시계 자세 테스트는 측정이 진행되는 동안 활성 테스트 자세를 사용자에게 명확히 표시해야 한다(shall). |
| FR-01-06 | 필수 | 시계 자세 테스트는 사용자에게 활성 테스트 자세와 연관된 측정 결과를 제공해야 한다(shall). |

## G04 · 다중 자세 시퀀스 디스플레이 (Multi-Position Sequence Display)

### 개요 (드래프트 문서)

Witschi Chronoscope X1 G3 사용 설명서(참고문헌 c)의 예시를 기반으로, 여러 시계 테스트 자세에 걸친 완전한 측정 주기를 지원하는 시퀀스 디스플레이 모드를 구현해야 한다. 이 디스플레이는 최대 10개 테스트 자세의 결과를 하나의 시퀀스로 캡처하고 검토할 수 있어야 한다. 각 자세별로 일차(rate), 진폭(amplitude), 비트 에러(beat error)의 측정 결과를 계산·표시하고, 시퀀스 전체에 걸친 요약 값도 산출해야 한다. 최소한 다음을 표시해야 한다:

- **X** — 모든 테스트 자세의 평균값
- **D** — 최대 측정값과 최소 측정값의 차이

또한 수직 자세와 수평 자세 간 비교를 지원하고, 해당되는 경우 밸런스 휠 불균형(balance-wheel unbalance)을 드러낼 수 있는 지표를 포함해야 한다. 이 디스플레이의 목적은 사용자가 자세별 시계 거동을 비교하고, 자세별 안정성과 일관성을 평가할 수 있도록 돕는 것이다. 특수 시계 유형 전용 기능이나 코드는 본 프로젝트에서 요구하지 않는다.

### 기능 요구사항 (FR)

| ID | 등급 | 요구사항 |

| FR-04-01 | 필수 | 다중 자세 시퀀스 디스플레이는 사용자에게 GUI 내에 시퀀스 디스플레이를 제공해야 한다(shall). |
| FR-04-02 | 필수 | 다중 자세 시퀀스 디스플레이는 사용자에게 여러 시계 테스트 자세에 걸친 완전한 측정 주기를 제공해야 한다(shall). |
| FR-04-03 | 필수 | 다중 자세 시퀀스 디스플레이는 사용자에게 하나의 시퀀스에서 최대 10개 테스트 자세의 결과를 캡처하고 검토할 수 있는 기능을 제공해야 한다(shall). |
| FR-04-04 | 필수 | 다중 자세 시퀀스 디스플레이는 사용자에게 최소 CH, CB, 6H, 9H, 3H, 12H 자세를 지원해야 한다(shall). |
| FR-04-05 | 필수 | 다중 자세 시퀀스 디스플레이는 사용자에게 각 측정 자세별 일차(rate), 진폭(amplitude), 비트 에러(beat error)를 제공해야 한다(shall). |
| FR-04-06 | 필수 | 다중 자세 시퀀스 디스플레이는 사용자에게 모든 유효 테스트 자세의 평균값 X를 제공해야 한다(shall). |
| FR-04-07 | 필수 | 다중 자세 시퀀스 디스플레이는 사용자에게 최대 측정값과 최소 측정값의 차이 D를 제공해야 한다(shall). |
| FR-04-08 | 권장 | 다중 자세 시퀀스 디스플레이는 사용자에게 수직 자세와 수평 자세 간 비교 기능을 제공하는 것이 바람직하다(should). |
| FR-04-09 | 권장 | 다중 자세 시퀀스 디스플레이는 사용자에게 밸런스 휠 불균형 가능성을 드러낼 수 있는 지표를 제공하는 것이 바람직하다(should). |

## G07 · 장기 성능 그래프 (Long-Term Performance Graph)

### 개요 (드래프트 문서)

시계의 일차(rate), 진폭(amplitude), 비트 에러(beat error)가 장시간에 걸쳐 어떻게 변화하는지를 기록·표시하는 장기 성능 그래프를 구현해야 한다. 이 그래프의 목적은 짧은 측정 중에는 보이지 않을 수 있는 변동 — 파워 리저브에 의한 변화, 날짜 변경 등의 컴플리케이션, 기타 주기적 거동 포함 — 을 사용자가 관찰할 수 있도록 돕는 것이다.

그래프는 테스트 중 주기적으로 갱신되어야 하고, 테스트 기간 전체의 전체 평균을 표시하며, 일반적인 변동 범위를 시각적으로 나타내어 사용자가 시계의 시간 경과에 따른 일관성을 빠르게 판단할 수 있어야 한다. 또한 경과 시간이 증가함에 따라 갱신 빈도를 줄여 장시간 테스트를 지원해야 하며, 이를 통해 여러 시간에 걸쳐 성능을 모니터링하면서도 가독성과 효율성을 유지할 수 있어야 한다.

|----|------|----------|
| FR-07-01 | 필수 | 장기 성능 그래프는 사용자에게 장시간에 걸친 시계의 일차(rate) 기록을 제공해야 한다(shall). |
| FR-07-02 | 필수 | 장기 성능 그래프는 사용자에게 장시간에 걸친 시계의 진폭(amplitude) 기록을 제공해야 한다(shall). |
| FR-07-03 | 필수 | 장기 성능 그래프는 사용자에게 장시간에 걸친 시계의 비트 에러(beat error) 기록을 제공해야 한다(shall). |
| FR-07-04 | 필수 | 장기 성능 그래프는 사용자에게 시간 경과에 따른 일차의 변화를 제공해야 한다(shall). |
| FR-07-05 | 필수 | 장기 성능 그래프는 사용자에게 시간 경과에 따른 진폭의 변화를 제공해야 한다(shall). |

| FR-07-07 | 권장 | 장기 성능 그래프는 사용자에게 테스트 중 그래프의 주기적 갱신을 제공하는 것이 바람직하다(should). |
| FR-07-08 | 권장 | 장기 성능 그래프는 사용자에게 전체 평균값을 제공하는 것이 바람직하다(should). |
| FR-07-09 | 권장 | 장기 성능 그래프는 사용자에게 변동 범위의 시각적 표시를 제공하는 것이 바람직하다(should). |
| FR-07-10 | 권장 | 장기 성능 그래프는 사용자에게 장시간 테스트 지원을 제공하는 것이 바람직하다(should). |
